# llmdev-reorient

## 何时使用

- 一段新对话开始、要继续之前的长任务。
- 怀疑上下文被压缩过，记忆和实际进度可能脱节。
- 任何长任务工作循环的第一步（参见 CLAUDE.md 中"重新定位协议"）。

## 怎么做

1. 定位本任务对应的文件：查 `docs/plans/PLAN-*.md` 和 `docs/state/STATE-*.md`。
   - 只有一组：直接用。
   - 有多组、且用户没指明是哪个任务：问用户是哪一个。
   - 一组都没有：说明这个任务还没建过账本，建议用 `llmdev-new-task` 先建。
2. 派发 `llmdev-state-checker` subagent，把确定好的 PLAN/STATE 文件路径
   传给它。让它去读文件内容、核对 git 真实状态、探测并跑测试，
   只把摘要带回来——不要自己在主对话里直接跑一堆 git/测试命令，
   那样会污染主窗口（这正是"善用 subagent 保持主窗口清爽"原则的落地）。
3. 拿到摘要后，对照 PLAN 判断：
   - STATE 记录的"已完成"和 git 真实状态是否一致？不一致以 git 为准，
     并更新 STATE 纠正记录。
   - 下一步该做什么？直接从摘要和 PLAN 的步骤列表里确定。
4. 用一两句话跟用户确认当前定位和下一步，然后继续执行。
