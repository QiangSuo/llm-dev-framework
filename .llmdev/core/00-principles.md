# 总纲：大模型开发的核心原则

> 这是工具无关的方法论核心。所有具体规则都从这里的心智模型推导而来。

## 核心心智模型：对话窗口是 RAM，磁盘是硬盘

对话窗口（context）不是模型的记忆，它是**易失的工作内存（RAM）**：

- 它有限，且会随着填满而退化（注意力被摊薄、早期内容被有损压缩）。
- 它随对话结束、崩溃、压缩而消失。
- 依赖它保存重要信息，等于把数据只存在 RAM 里不落盘。

真正的记忆是**文件系统（硬盘）**：plan、决策、进度、账本都该落成文件。

由此推出一条贯穿全框架的铁律：

> **任何重要的东西，都必须在它离开窗口之前，已经写进磁盘。**

做到了这一点，"换对话"和"上下文被压缩"就都变成廉价、可生还的操作——
因为窗口里从不存在"只活在窗口里"的关键信息。

## 三条总原则

1. **状态外化（Externalize state）**
   开发的真实状态活在磁盘上（代码、plan、STATE 账本、日志），不活在对话里。

2. **小步可验证（Small verifiable steps）**
   把工作切成小到能在一个上下文预算内干净做完的步骤，每步都有明确的验证。

3. **从产物重新定位（Re-derive position from artifacts）**
   要知道"我在哪"，靠读磁盘上的真实状态（git、测试、文件），而不是靠回忆对话。

## 两类失败，两套药

- **记忆问题**（失忆、丢上下文）→ 用状态外化 + 重新定位解决。见 `10-context-and-memory.md`。
- **行为问题**（跑偏、空转、假装成功）→ 用卡住协议 + 边界 + 验证诚信解决。见 `20-long-horizon.md`。
